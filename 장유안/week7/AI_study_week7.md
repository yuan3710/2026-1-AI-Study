DNABERT: pre-trained Bidirectional Encoder Representations from Transformers model for DNA-language in genome



<들어가기에 앞서>

* BERT = 인간의 언어를 컴퓨터가 이해할 수 있도록한 자연어 처리 모델 (양방향 단어 처리, 사전학습 + 미세조정)
* DNA분석을 하는 이유 = 우리 몸의 설계도인 DNA를 읽어서 생명 현상의 원리를 파악한다.



###### **Abstract**

* DNABERT = DNA서열을 언어처럼 보고 BERT로 학습한 모델 (사전학습 양방향 인코더)
* 서로 같은 DNA조각이더라도 상황에 따라 다른 역할을 할 수 있고(다의성), DNA의 멀리 떨어진 부분도 서로 영향을 준다. (인간의 자연어와 유사)
* 차별점1 : DNABERT는 입력 DNA에서 각 염기(nucleotide)의 중요도와 의미적 관계를 직접 시각화할 수 있어 모델 해석이 가능
* 차별점2: 인간 유전체로 사전학습한 DNABERT가 다른 생물종에도 매우 우수한 성능으로 적용될 수 있음

###### 

###### **Introduction**

* DNA중 단백질 형성을 담당하지 않는 비암호화(non-coding) DNA를 해석하는 유전체 연구를 효율적으로 하기 위해 자연어 처리 모델인 BERT의 도입이 제시됨
* A, T, G, C의 글자 배열이 어떤 단백질로 번역되는지를 나타내는 유전 암호(genetic code)는 모든 생물에서 공통적.
* 유전자가 언제 어떻게 발현될 지를 정하는 조절 코드(regulatory code)는 세포종류, 생물종마다 다름
* CRE = cis- regulatory code
* 유전자 발현 ->  '문맥'이 중요= 어떤 세포인지, 어떤 종인지
* 이전의 CNN, RNN 모델들은 대량의 라벨 데이터가 필요한데 생체 데이터 특성상 구하기 어려움



###### **Materials and method**

* 따라서, DNABERT는 대부분 Attention 기반 Transformer구조를 사용
* DNA 딥러닝 -> 라벨이 없는 대량의 인간의 유전체만으로도 학습 가능, 스스로 중요한 부분에 집중 가능
* DNABERT의 진행 단계
* 1\. DNA서열을 k-mer토큰으로 변환 (k개 단위로 하나로 묶어 취급)(여러 염기를 이어붙인 것을 의미)
* 2\. 각 토큰을 벸터로 바꾼다.(수식화)
* 3\. 모든 토큰끼리의 관계를 계산한다.
* 3-2. 다양한 attention head에서 다른 관점으로 DNA해석
* 4\. 결과 업데이트



###### **Training**

* 인간의 DNA를 이루는 염기서열은 바로 앞뒤만 가리면 유추가 쉽기에 기존 BERT모델보다 더 넓은 범위 가리고 학습 진행
* 다량의 학습이 필요하고 시간이 오래 걸리기에 해당 논문의 저자들이 데이터 배포
* 120000step까지 진행이후 loss감소 확인 불가



###### **Result**

* 유전자를 읽기 시작한다는 것을 알려주는 역할의 Promoter
* Promoter를 빠르게 찾는 것이 DNA분석 모델의 중요 포인트
* 해당 측면에서 DNABERT는 CNN기반의 기존 모델보다 뛰어난 성능을 증명
* p53은 대표적인 암 억제 유전자
* DNABERT로 진행한 실험에서 기타 구분이 어려운 유사한 유젅자들 사이에서 성공적으로 p53을 찾아냄
* \-> 암 연구에 있어 큰 도움이 될 것
* 단순힝 표면적인 패컨만 외운 것이 아니라 주변 DNA 문맥과 내부의 조절 신호도 이해했기 때문에 어려운 과제에서도 높은 성능 유지 가능
* 유전자 변이 학습
* 1\. DNABERT가 중요하다고 한 변이
* 2\.실제 데이터베이스와 비교
* 3\. 관련 질병인지 여부 확인
* 4\. 유전자 변이가 어떤 질병으로 이어지는지 Attention 모델을 통해 확인
